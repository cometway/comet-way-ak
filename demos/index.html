In order to run these demo agents, you must use the command console. Go to the demo subdirectory of the demo you want to run and type ak in the command console. For those of you not fimilar with using the command console, we will step you thorugh the WebServer Demo beacuse it is easy and impressive.

Webserver

 This demo starts a webserver on your machine and serves documentation and demos of the Comet Way Web Application Server. Open the command prompt console (usually located in the Acceseries Folder in Program Files). Type the following command to change to the correct directory. 

	
[~/ak] % cd  ak/demos/webserver
[~/ak/demos/webserver] % _

This agent scans a directory for new and updated files based on their file date. It uses the scheduler to wakeup every 15 seconds.

Email
This demo retrieves email from a POP3 email box, sends email through a SMTP server, auto-replys to ReceiveEmailAgent at the specified POP3 sever. Before this demo will work correctly, these startup files will need to be edited to reflect your preferred (functional) email settings. To edit the staerup files using the Agent Browser, open a command prompt to the email subdirectory and enter AB command.

Reporters
There are variety of different Reporters that can be used with the Agent Kernel. Some Reporters can be specified as option for the AK command, and others have to be started as Agents.

To disable agent output completely:
ak -reporter null 200_SampleAgent.agent

To view the output in a VT100 console, the VT100ConsoleReporter highlights debug messages in bold. Warnings and errors are inversed.
ak -reporter com.cometway.io.VT100ConsoleReporter 200_SampleAgent.agent

Send output to log files using the FileReporter:
ak 100_FileReporter.agent 200_SampleAgent.agent

Reporters can be written to manage and route agent output in any number of methods that are appropriate. It is definitely possible to write your own Reporter Agents that log activity to a database or TCP/IP connection if desired.

Robot

Sample Agent
This demonstrates some of the more common agent methods such as println, debug, warning, and error methods, as well as some examples of using the agent Props to store and retrieve data.

Scheduler
This demo shows how to write an agent that uses the Scheduler to wakeup every 2 seconds.

Sessions
In this demo, an ObjectCache is started along with a SessionManager to provide a session management service. The Scheduler wakes up the SessionTest every 5 seconds which uses the SessionManager to create a session ID. The SessionReaper is scheduled to remove sessions that have been dormant for 30 seconds.

System Properties
This demo prints the java.lang.System properties to the console.

XML Weather Demo
A demo on an XML webserver. This is a webservice that retreives the weather online and reports the results back in XML. You must have an Internet connection for this demo is run successfully. Once you start the demo, go to xml_weather\index on your local machine.

Tutorial for Hello, World! (Agent Anatomy 101)

Overview
This tutorial starts out with the venerable "Hello, world" example. The HelloWorldAgent is a simple example of the correct form for writing an agent.

import com.cometway.ak.*;

/** Everyone's favorite written as an agent. */

public class HelloWorldAgent extends Agent
{
	public void start()
	{
		println("Hello, world!");
	}
}

All agents must import com.cometway.ak.*.

All agents must extend com.cometway.ak.Agent, any of its subclasses, or implement com.cometway.ak.AgentInterface when subclassing is not feasible.

All agents must provide a constructor that accepts a com.cometway.props.Props as its argument. When extending Agent or any of its subclasses, this argument can be passed to the parent class constructor using super. Typically no initialization occurs in the agent constructor; instead the start method is used for initialization routines. The constructor exists so that the Agent Kernel can associate a set of attributes or Props with the newly created agent instance.

All agents must provide a start() method, or extend a subclass of Agent that provides one. The HelloWorldAgent provides its own start method which prints the message "Hello, world!" to the console before exiting.

Compiling HelloWorldAgent
The following output depicts using the included jc script to compile HelloWorldAgent from the command-line: 

[localhost:ak/examples/src] cometdev% jc HelloWorldAgent.java
Compiling...
HelloWorldAgent.java
[localhost:~/projects/ak/examples] cometdev%

The jc script is a convenience tool that makes it easier to compile Java source code and have the resulting class files routed appropriately to a separate directory on the CLASSPATH. This separation of classes from source code makes it easier to manage your files. The jc script is located in the ak/bin/ directory of this release. The command-line equivilant to jc is similar to: 

cometdev% javac -d ~/classes -classpath ~/classes HelloWorldAgent.java

Configuring a Props File for HelloWorldAgent
Before HelloWorldAgent can be started, it needs a Props file listing attributes that make up the agent's identity. The only required attribute (or property) is classname. This property is set to the fully qualified classname of the agent, in this case HelloWorldAgent. The agent Props file is a standard text file containing entries like the following example: 

classname=HelloWorldAgent

Both the ak/agents/ and ak/examples/ directories contain Props files for a variety of agents included with this release. Each Props filename typically ends with the .agent extension. The following output depicts using the ak script to bootstrap the Agent Kernel and start the HelloWorldAgent:

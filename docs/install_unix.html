Installation Instructions for UNIX

Before You Install
Proper installation of the Comet Way Agent Kernel requires a working knowledge of the Java platform upon which it will be deployed. Instructions in this document are generic in nature, but refer specifically to most common UNIX based implementations of the JDK. In most cases, you will need to be able to answer the following Java-related questions:

Where is Java installed?

You will need to know this if you are using JDK 1.1.x, or find that the JDK tools need to be added to your PATH. If Java is not installed at all, you can begin by downloading one of the available reference implementations by visiting the J2SE Downloads Page on the java.sun.com website.

Are the JDK tools on your PATH?

For convenience sake, it is a good idea to have the JDK tools directory on your PATH. If you know where Java is installed, you will find the JDK tools in the bin/ subdirectory. Some implementations of UNIX (ie: Solaris, SuSE, Mac OS X) may have this setup by default.

Is your Java CLASSPATH set?

If you are using JDK 1.1.x you may need to set your CLASSPATH environment variable to include the standard Java classes. If you know where Java is installed, you will need to add the lib/classes.zip file to your CLASSPATH. With JDK 1.2 and higher this is no longer necessary.

Do you know how to use the java and javac tools?

It is essential to understand how to use java and javac if you are going to write Java agents in a command line environment like UNIX.

If you are unexperienced with Java development in general, we highly recommend reading the first three chapters of Java In A Nutshell, published by O'Reilly. This book contains most of the necessary information critical to learning how to write, compile, and execute applications written in the Java language.

Installation Instructions

Check the download file size

After you have finished downloading the tar.gz archive from Comet Way Downloads, check the file size to make sure the file has downloaded completely.

Decompress the tar.gz archive

From the UNIX command line, decompress the tar.gz archive into the location where you intend to install Comet Way Agent Kernel. 

[mario:~] % ls
ak-02-11-2002.tar.gz
[mario:~] % gunzip ak-02-11-2002.tar.gz 
[mario:~] % tar xf ak-02-11-2002.tar 
[mario:~] % ls
ak                ak-02-11-2002.tar
[mario:~] % _

Most modern UNIX implementations also support tar xfz which does the gunzip and tar xf in one step, leaving the original tar.gz file intact.

Remove the downloaded file (optional)

To conserve disk space, you may delete the download-related archives from the file system. 

[mario:~] % rm ak-02-11-2002.tar
[mario:~] % _

Run the install.sh script

We have provided a simple installation script that:
  1. Creates a ~/bin directory (if you don't have one).
  2. Creates a ~/classes directory (if you don't have one).
  3. Creates the ~/bin/jc script.
  4. Creates the ~/bin/ak script.
  5. Creates the ~/bin/ab script.
  6. Copies the demo classes to your class repository (~/classes). 

[mario:~] % cd ak
[mario:~/ak] % ls
LICENSE.html ak.jar       classes      docs         install.sh   src
README.html  bin          demos        import       javadocs
[mario:~/ak] % ./install.sh 
Installing Comet Way Agent Kernel Scripts
Creating /Users/mario/bin directory
Creating /Users/mario/classes directory
Creating /Users/mario/bin/jc
Creating /Users/mario/bin/ak
Creating /Users/mario/bin/ab
Copying Demo classes to /Users/mario/classes
[mario:~/ak] % _

Test the Agent Kernel
OK, now it's time to see if the Agent Kernel is installed properly. The Agent Kernel can be started from any directory by issuing the ak command. If you are using csh, tcsh, or zsh you will first need to rehash your command completion buffer by using the rehash shell command. 

[mario:~/ak] % rehash
[mario:~/ak] % ak
[AK] Comet Way Agent Kernel 2.1.2 10-31-2003
[000_AgentKernel] Starting on 2003/10/30 17:28:49.573 EST
[000_AgentKernel] Creating agent com.cometway.xml.XMLStartupAgent
[001_XMLStartupAgent] Starting on 2003/10/30 17:28:49.603 EST
[001_XMLStartupAgent] Loading agents from ak.xstartup
[001_XMLStartupAgent] Loading agents from .
[mario:~/ak] % _

Note: You do not need to use the 'rehash' command unless you are using a csh shell variant. Common csh variants are tcsh and zsh.

You can issue the ak command in any directory that contains .startup files. Try issuing the ak command in the demos/hello_world directory. 

[mario:~/ak] % cd demos/hello_world
[mario:~/ak/demos/hello_world] % ak
[AK] Comet Way Agent Kernel 2.1.2 10-31-2003
[000_AgentKernel] Starting on 2003/10/30 17:30:36.187 EST
[000_AgentKernel] Creating agent com.cometway.xml.XMLStartupAgent
[001_XMLStartupAgent] Starting on 2003/10/30 17:30:36.207 EST
[001_XMLStartupAgent] Loading agents from ak.xstartup
[000_AgentKernel] Creating agent HelloWorldAgent
[100_HelloWorldAgent] Starting on 2003/10/30 17:30:36.337 EST
[100_HelloWorldAgent] Hello, world!
[mario:~/ak] % _

If you follow the output displayed in your console, you can see the the XMLStartupAgent found an .xstartup file in current dir called ak.xstartup. It then created and started the HelloWorldAgent. The HelloWorldAgent outputs the message "Hello, world!".

Now try some of the other demos!!!

Contents of This Release
This release contains the following files and directories:

README.html Primary documentation and table of contents for this releease.
LICENSE.html This contains the open source license for Comet Way Agent Kernel.
ak.jar This Java Archive (JAR file) contains all of the classes necessary to compile and run any of the kernel, web, or email agents. It does not contain classes for any of the examples; they must be compiled separately.
install.sh This installer script can be used to setup the Comet Way Agent Kernel for use in your home directory.
bin/ Contains UNIX shell scripts for recompiling the ak.jar file and creating javadocs, a script for compiling individual Java classes to a class repository (jc), a tool for starting the Agent Kernel (ak), and a Swing-based tool for browsing and changing agent startup configurations (ab).
classes/ This is a class repository containing class files for the demos.
demos/ Contains agent code examples and their associated Props files.
docs/ Contains HTML-based documentation about the Comet Way Agent Kernel.
import/ Contains 3rd party open source libraries that are used by the agents.
javadocs/ Complete HTML-based documentation for all of the classes in this release.
src/ Contains the complete Java source code for the kernel, web, and email, and other agents in this release.
